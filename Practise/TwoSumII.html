Two Sum Algorithm Visualizer - Two Pointers Approach

This visualization demonstrates how the Two Sum algorithm works using the two pointers approach on a sorted array.

How the Algorithm Works

  • The algorithm uses two pointers: one at the start (i) and one at the end (j) of the sorted array
  • It calculates the sum of the elements at the two pointers
  • If the sum equals the target, the solution is found
  • If the sum is less than the target, the left pointer moves right (i++)
  • If the sum is greater than the target, the right pointer moves left (j--)
  • This approach has O(n) time complexity and O(1) space complexity

Algorithm Code

public int[] twoSum(int[] numbers, int target) {
int i = 0;
int j = numbers.length - 1;
while (i < j) {
int sum = numbers[i] + numbers[j];
if (sum == target) {
return new int[]{i + 1, j + 1};
} else if (sum < target) {
i++;
} else {
j--;
}
}
return null;
}

Time & Space Complexity Analysis

Time Complexity

O(n)

The algorithm iterates through the array at most once, with each step moving one of the pointers.

In the worst case, it will check n-1 elements before finding the solution.

Space Complexity

O(1)

The algorithm only uses a constant amount of extra space for the two pointers and the sum variable.

This is more space-efficient than the HashMap approach.
