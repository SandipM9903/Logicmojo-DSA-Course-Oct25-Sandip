Two Sum Algorithm Visualizer

This visualization demonstrates how the Two Sum algorithm efficiently finds two numbers that add up to a target value using a HashMap.

How the Algorithm Works

  • The algorithm uses a HashMap to store numbers and their indices as it iterates through the array
  • For each number, it calculates the complement (target - current number)
  • If the complement exists in the HashMap, the solution is found
  • Otherwise, the current number and its index are added to the HashMap
  • This approach has O(n) time complexity and O(n) space complexity

Algorithm Code

public int[] twoSum(int[] nums, int target) {
Map map = new HashMap<>();
for(int i = 0; i < nums.length; i++) {
int need = target - nums[i];
if(map.containsKey(need)) {
return new int[]{i, map.get(need)};
}
map.put(nums[i], i);
}
return null;
}

Time & Space Complexity Analysis

Time Complexity

O(n)

The algorithm iterates through the array once, performing constant-time operations (HashMap lookups and inserts) for each element.

HashMap operations (containsKey, get, put) have O(1) average time complexity.

Space Complexity

O(n)

The algorithm uses a HashMap that can store up to n elements in the worst case.

This is the trade-off for achieving O(n) time complexity instead of O(n²).
